        How onebrain handles your upload

        What happens to a file you upload

        1. onebrain extracts the text — from PDFs, Word/Excel/PowerPoint, plain text, or images (via OCR).
        2. The text is split into small chunks, turned into embeddings, and stored.
        3. When someone asks a question, onebrain first filters the chunks down to only the ones that person's role may see, then retrieves the most relevant few.
        4. Only those allowed chunks are sent to the AI. The model never sees a chunk the asker isn't cleared for — access is enforced in code and the database, not by asking the model nicely.

        The label you set decides who can retrieve it

        A chunk is returned only if the asker clears all three — classification, location, and category — within their own organisation.

        Classification — how sensitive

        • Public Anyone, including signed-out customers.
        • Internal Any signed-in employee.
        • Confidential Location managers, marketing, and above.
        • Restricted Only the owning department (e.g. HR, Finance) and admins.

        Location — which site it belongs to

        Global is visible to everyone (subject to classification). A specific gym (Munich / Berlin / Hamburg) is visible only to staff at that gym — chain-wide roles like HR, Finance and Admin still see it.

        Category — which department compartment

        A role sees a compartment only if it's on its list. General is open to everyone (subject to classification); the rest (Customer service, Operations, HR, Finance, Marketing) are walled off to the departments that own them.

        Supported file types

        PDF (with OCR for scanned pages), Word .docx, Excel .xlsx/.xlsm, PowerPoint .pptx, RTF, images via OCR (.png .jpg .tiff …), and any text format (.txt .md .csv .json .html …).
